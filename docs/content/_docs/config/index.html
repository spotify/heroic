This section will go through the available configuration options in detail.

Heroic is divided up into a few components, each whose configuration will affect the service in different ways. Each component has its own sub-section, which are the following.

Cluster

This section documenta how to configure clustering and federation to support global interfaces.

Metrics

This section describes how to configure persistent storage of metrics.

Metadata

This section documents how metadata backends are configured. Metadata backends are responsible for indexing the active time series to support filtering.

Suggest

This section documents how suggestion backends are configured. Suggestions is a feature that provides the user with feedback on which tags and time series are available.

Elasticsearch Connections

This section documents how common Elasticsearch connections are configured. They are typically used in metadata and suggestion backends that uses Elasticsearch.

Query logging

This section documents how to configure query logging. I.e. detailed logging of internal Heroic state at different stages of query processing.
